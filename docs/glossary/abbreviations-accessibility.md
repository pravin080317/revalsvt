# Abbreviations and Accessibility Glossary

Purpose: ensure abbreviations are expanded on first use and understandable for all users.

## Core abbreviations

- `QC`: Quality Control
- `PAD`: Property Attribute Details
- `SVT`: Sales Verification Tool
- `VOA`: Valuation Office Agency

## Recommended first-use pattern

Use the full term first, then the abbreviation in brackets.

Examples:
- `Quality Control (QC)`
- `Property Attribute Details (PAD)`
- `Sales Verification Tool (SVT)`

## Suggested wording updates for current UI labels

Use these as content updates when implementing accessibility fixes.

- `Submit Sales Verification Task for QC` -> `Submit Sales Verification Task for Quality Control (QC)`
- `Submit for QC` -> `Submit for Quality Control (QC)`
- `QC undertaken by` -> `Quality Control (QC) undertaken by`
- `QC outcome` -> `Quality Control (QC) outcome`
- `QC remarks` -> `Quality Control (QC) remarks`
- Tooltip `QC review of the caseworker's sales verification decision` -> `Quality Control (QC) review of the caseworker's sales verification decision`
- `PAD Confirmation` -> `Property Attribute Details (PAD) confirmation`
- `Select PAD confirmation` -> `Select Property Attribute Details (PAD) confirmation`

## Accessibility note

Do not rely only on `title` tooltips to explain abbreviations. Keep key expansions visible in labels or nearby text so keyboard, touch, and assistive technology users receive the same information.
